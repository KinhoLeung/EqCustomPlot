An interactive, Qt-based equalizer (EQ) response visualizer and editor, built on top of QCustomPlot. It demonstrates how to compose multiple biquad filters, display the overall frequency response in real time, and provide a polished UX for editing band parameters.

This project targets Qt 6.5+ and uses the following Qt modules: Core, Widgets, Svg, PrintSupport. It also includes a custom axis ticker for logarithmic frequency axes.


## Key Features
- Interactive multi-band EQ editor with 11 bands: High-Pass, 9 param bands (1–9), and Low-Pass.
- Realtime combined frequency response curve with per-band curves.
- Logarithmic frequency axis (approximately 16 Hz – 25 kHz) and amplitude range −18 dB … +18 dB.
- Context menu per band:
  - Filter Type: Peak, LowShelf, HighShelf, LowPass, HighPass, BandPass, Notch, AllPass
  - Bypass per band
  - Solo a single band (temporarily bypass others)
- Themed SVG icons (dark/light) loaded via Qt Resource System.
- High-quality plotting via QCustomPlot with antialiasing and custom grid.

## Build Requirements
- Qt 6.5 or newer
  - Components: Core, Widgets, Svg, PrintSupport
- CMake ≥ 3.19
- A C++17-capable compiler

The project is configured with CMake in `CMakeLists.txt` and builds a single executable `EqCustomPlot`.


## Build & Run (CMake)
```bash
# from project root
cmake -S . -B build -DCMAKE_PREFIX_PATH=<path-to-Qt/6.x.x>/<compiler> \
      -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release

# On Windows you may find the executable in:
#   build/Release/EqCustomPlot.exe
# On Linux/macOS:
#   build/EqCustomPlot

# Optionally, install/deploy (uses qt_generate_deploy_app_script)
cmake --install build --config Release
```
Notes:
- If Qt is not in your default CMake prefix path, set `CMAKE_PREFIX_PATH` to your Qt installation (e.g. `C:/Qt/6.5.3/msvc2019_64`).
- The project enables `CMAKE_AUTORCC` to compile `resources.qrc` automatically.


## Usage
- Launch the app. The central widget is an `EqCustomPlot` instance.
- Click a band (icon) to select it; drag to adjust frequency and gain.
- Use the context menu (right-click on a band) to:
  - Toggle `Bypass`
  - Toggle `Solo`
  - Change `Filter Type`
- The top/overall curve shows the sum of all active filters; each band has its own curve.


## Project Structure
- `main.cpp`: Qt entry point.
- `mainwindow.*`: Creates a `QMainWindow`, embeds `EqCustomPlot` as the central widget.
- `eqcustomplot.*`: The interactive plot logic; context menus, icons, band management, painting.
- `qcpaxistickerfreq.*`: Custom non-uniform axis ticker for log-frequency labeling and ticks.
- `eq.*`: Biquad filter math. Includes:
  - `getSectionsMatrix(...)`: Compute filter SOS coefficients per band.
  - `getFreqzn(...)`: Compute magnitude response for one or multiple sections.
- `qcustomplot.*`: QCustomPlot source (vendored).
- `resources.qrc`: Qt resource file for themed SVG icons under `image/`.
- `image/`: Dark/Light SVG assets for band icons.
- `CMakeLists.txt`: Build configuration.


## Technical Notes
- Frequency axis is logarithmic; tick labels are generated by `QCPAxisTickerFreq` with decade-aware rounding (e.g., 1k/2k/5k).
- Y axis uses `QCPAxisTickerText` to display evenly spaced dB labels.
- Default ranges and steps:
  - X: 16 … 25,000 (log scale)
  - Y: −18 … +18 dB
  - Sample Rate: 384,000 Hz
  - Frequency grid resolution: 1,024 points
- Each band stores defaults and runtime properties via `QCPGraph::setProperty` (e.g., `defaultQ`, `defaultFreq`, `type`, `bypass`).
